# Sightglass benchmark runner

Canonical reference: `notes/test_commad_cheetsheet.md` in the repo. Always prefer that file if it has been updated since this skill was written.

## Preconditions

- Debug build of `wasmedgeIRBenchmarkTests` exists in `build/test/ir/` (build with `make wasmedgeIRBenchmarkTests -j32` from `build/`).
- Test data at `test/ir/testdata/sightglass/` (upstream kernels) and/or `test/ir/testdata/sightglass-strong/` (strengthened goldens — see below).

## `sightglass` vs `sightglass-strong`

Two kernel directories live under `test/ir/testdata/`:

- **`sightglass/`** — upstream kernels. Goldens (`*.stdout.expected` / `*.stderr.expected`) were generated by WasmEdge itself, so a regression in WasmEdge's own loader/interpreter/JIT paths can silently corrupt both the runtime output and the oracle — the diff will still look clean. Good for perf/metrics, weak for correctness.
- **`sightglass-strong/`** — same `.wasm` kernels, but goldens were regenerated by an **independent** runtime (wasmtime 43.0.1, via `bench_stub.wat` shim — see `testdata/sightglass-strong/ORACLE.md` for provenance). Use this whenever you care about correctness: a real regression in WasmEdge's IR JIT / codegen / runtime will show up as a golden mismatch that `SightglassSuite`'s `checkExpected` will flag.

### How to switch

The harness reads `WASMEDGE_SIGHTGLASS_DIR` (see `ir_benchmark_test.cpp` around L1302). A relative path is resolved against `test/ir/testdata/`; an absolute path is used as-is. The kernel-discovery loop in the skill's commands iterates the filesystem directly, so **also update the glob** when running the strong set.

### Canonical command: perf run on the strong set, IR JIT O2

```shell
cd build && \
for wasm in ../test/ir/testdata/sightglass-strong/*.wasm; do
  kernel="$(basename "$wasm" .wasm)"
  echo "Testing $kernel:"
  WASMEDGE_SIGHTGLASS_DIR=sightglass-strong \
  WASMEDGE_SIGHTGLASS_KERNEL="$kernel" \
  WASMEDGE_SIGHTGLASS_MODE=IR_JIT \
  WASMEDGE_SIGHTGLASS_QUICK=1 \
  WASMEDGE_IR_JIT_OPT_LEVEL=2 \
  WASMEDGE_IR_JIT_BOUND_CHECK=0 \
  stdbuf -oL timeout 30 ./test/ir/wasmedgeIRBenchmarkTests --gtest_filter='*SightglassSuite*' 2>&1
done | tee /tmp/wasm-strong-test.log 2>&1
echo $?
grep -iE 'dumped|error|failed|mismatch|warning' /tmp/wasm-strong-test.log
```

The final `grep` is required by the repo's testing rules — a stdout/stderr mismatch against the strong goldens will show up as a gtest `EXPECT_EQ` failure line, which the grep catches even if the outer exit code is 0.

### When to use which

- **Perf runs** (collecting metrics, comparing tier-2 configs, opt-level sweeps) → **sightglass-strong**.
- **Correctness checks** (validating a new pass, regression-gating before a commit, stdout/stderr golden diffs) → **sightglass** (upstream, normal set).
- Both sets contain the same kernel names, so `WASMEDGE_SIGHTGLASS_KERNEL=<name>` works identically — only `WASMEDGE_SIGHTGLASS_DIR` (and the shell loop's glob) changes.

## Rules

1. **Never run Interpreter mode for benchmarks** — too slow. Default to `WASMEDGE_SIGHTGLASS_MODE=IR_JIT` unless the user explicitly asks for another mode.
2. When `MODE` is set, the `SKIP_*` flags are redundant.

## Canonical commands

### Single kernel, IR JIT O2

```shell
cd build && WASMEDGE_SIGHTGLASS_MODE=IR_JIT WASMEDGE_IR_JIT_OPT_LEVEL=2 WASMEDGE_SIGHTGLASS_KERNEL=<name> timeout 30 ./test/ir/wasmedgeIRBenchmarkTests --gtest_filter='*SightglassSuite*'
```

### All kernels, IR JIT only

```shell
cd build && \
for wasm in ../test/ir/testdata/sightglass/*.wasm; do
  kernel="$(basename "$wasm" .wasm)"
  echo "Testing $kernel:"
  WASMEDGE_SIGHTGLASS_KERNEL="$kernel" \
  WASMEDGE_SIGHTGLASS_MODE=IR_JIT \
  WASMEDGE_IR_JIT_OPT_LEVEL=2 \
  stdbuf -oL timeout 30 ./test/ir/wasmedgeIRBenchmarkTests --gtest_filter='*SightglassSuite*' 2>&1
done | tee /tmp/wasm-test.log 2>&1
```

### All kernels with tier-2 enabled (aggressive thresholds)

```shell
cd build && \
for wasm in ../test/ir/testdata/sightglass/*.wasm; do
  kernel="$(basename "$wasm" .wasm)"
  echo "Testing $kernel:"
  WASMEDGE_SIGHTGLASS_KERNEL="$kernel" \
  WASMEDGE_SIGHTGLASS_MODE=IR_JIT \
  WASMEDGE_IR_JIT_OPT_LEVEL=2 \
  WASMEDGE_TIER2_ENABLE=1 \
  WASMEDGE_TIER2_THRESHOLD=10 \
  WASMEDGE_TIER2_LOOP_THRESHOLD=5 \
  stdbuf -oL timeout 60 ./test/ir/wasmedgeIRBenchmarkTests --gtest_filter='*SightglassSuite*' 2>&1
done | tee /tmp/wasm-tier2-test.log 2>&1
```

## Env var quick reference

Sightglass harness: `MODE` (Interpreter/JIT/IR_JIT/AOT), `DIR`, `KERNEL`, `QUICK`, `SKIP_INTERP`, `SKIP_LLVM_JIT`, `SKIP_AOT`, `WASMEDGE_QUIET`.

IR JIT: `WASMEDGE_IR_JIT_OPT_LEVEL` (0/1/2, default 2), `WASMEDGE_IR_JIT_DUMP` (writes `/tmp/wasmedge_ir_NNN_{before,after}.ir`), `WASMEDGE_IR_JIT_BOUND_CHECK`.

Tier-2: `WASMEDGE_TIER2_ENABLE=1` (only `1` works), `THRESHOLD` (default 10000), `LOOP_THRESHOLD` (default 1000), `MAX_COMPILE`, `TRACE_FUNC`, `DUMP_IR=<dir>`.

## Debugging a JIT crash

Use a Debug build. `wasm_jit_NNN` = (NNN+1)th JIT'd function.

```shell
cd build
WASMEDGE_SIGHTGLASS_MODE=IR_JIT WASMEDGE_SIGHTGLASS_KERNEL=<name> WASMEDGE_IR_JIT_OPT_LEVEL=2 \
  gdb --args ./test/ir/wasmedgeIRBenchmarkTests --gtest_filter='*SightglassSuite*'
```

In GDB on SIGSEGV: `x/i $pc`, `disas $pc-32,$pc+32`, `info registers`. Break at `WasmEdge::VM::IRJitEngine::invoke` to identify which kernel/function is running.

For IR inspection, set `WASMEDGE_IR_JIT_DUMP=1` and inspect `/tmp/wasmedge_ir_NNN_{before,after}.ir` (CFG annotations on the `after` file).
