# Sightglass benchmark runner

**Commands live in `notes/test_commad_cheetsheet.md`** — refer to that file for every invocation (single kernel, full sweep, tier-2, tier-2+OSR, GDB). This skill carries the *rules*; the cheatsheet carries the *commands*.

## Preconditions

- Debug build of `wasmedgeIRBenchmarkTests` in `build/test/ir/` (`make wasmedgeIRBenchmarkTests -j32` from `build/`).
- Test data at `test/ir/testdata/sightglass/` and/or `test/ir/testdata/sightglass-strong/`.

## `sightglass` vs `sightglass-strong`

- **`sightglass/`** — upstream kernels; goldens generated by WasmEdge itself. A regression in WasmEdge's own loader/interp/JIT can silently corrupt both runtime output and oracle (the diff stays clean). Good for perf, weak for correctness.
- **`sightglass-strong/`** — same `.wasm` kernels; goldens regenerated by an independent runtime (wasmtime 43.0.1, see `testdata/sightglass-strong/ORACLE.md`). Real WasmEdge regressions surface as golden mismatches.

When to use which:

- **Perf runs** (metrics, tier-2 sweeps, opt-level sweeps) → **sightglass-strong**.
- **Correctness checks** (validating a new pass, regression-gating, golden diffs) → **sightglass** (upstream, self-consistent goldens avoid false positives).

Both sets contain the same kernel names; only `WASMEDGE_SIGHTGLASS_DIR` and the loop's glob change.

## Rules

1. **Never run Interpreter mode for benchmarks** — minutes per kernel, useless signal. Default `WASMEDGE_SIGHTGLASS_MODE=IR_JIT` unless the user explicitly asks for another mode.
2. When `MODE` is set, the `SKIP_*` flags are redundant.
3. **OSR requires tier-2.** Without `WASMEDGE_TIER2_ENABLE=1` the OSR continuation is never compiled — sweeps that omit tier-2 don't actually verify OSR (the back-edge diamond polls a slot that never gets published).
4. **Always grep the log after a run** (CLAUDE.md rule #4): `grep -iE 'dumped|error|failed|mismatch|warning' <log>`. Exit 0 alone is not success — gtest can return 0 while per-kernel mismatches or `core dumped` lines sit in the log.

## Env var headlines

Full descriptions and defaults in the cheatsheet.

- **Harness**: `MODE`, `DIR`, `KERNEL`, `QUICK`, `SKIP_INTERP`, `SKIP_LLVM_JIT`, `SKIP_AOT`, `WASMEDGE_QUIET`.
- **IR JIT**: `WASMEDGE_IR_JIT_OPT_LEVEL` (0/1/2, default 2), `WASMEDGE_IR_JIT_DUMP`, `WASMEDGE_IR_JIT_BOUND_CHECK`.
- **Tier-2**: `WASMEDGE_TIER2_ENABLE=1`, `THRESHOLD`, `MAX_COMPILE`, `TRACE_FUNC`, `DUMP_IR`.
- **OSR**: `WASMEDGE_OSR_THRESHOLD` (`0` disables, recommended 1000+), `OPT_LEVEL`, `MIN/MAX_FUNC`, `MIN/MAX_LOOP`, `SKIP_STORES` (debug only — reads stale locals), `DUMP`.

## Debugging a JIT crash

Use a Debug build. `wasm_jit_NNN` = (NNN+1)th JIT'd function (registered via `ir_gdb_register`). Full GDB workflow in the cheatsheet's "Debugging with GDB" section. For IR inspection set `WASMEDGE_IR_JIT_DUMP=1` and read `/tmp/wasmedge_ir_NNN_{before,after}.ir` (CFG annotations on the `after` file).
